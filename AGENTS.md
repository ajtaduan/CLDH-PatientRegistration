# AI Assistance Disclosure

This project was built with AI assistance (Claude) as allowed under the assessment's
AI-Assisted Programming Policy. This document explains how it was used.

## How AI was used
- Explaining ASP.NET Core / EF Core concepts I hadn't used before (I have prior
  experience with PHP/MySQL and JavaScript, but this was my first ASP.NET Core project)
- Generating boilerplate and initial code for models, controllers, auth setup, and
  the frontend (HTML/CSS/JS)
- Debugging build errors and JS bugs as I typed the code myself
- Reviewing decisions (e.g. cookie auth vs JWT, single seeded admin account vs
  full registration flow) against the assessment requirements

## What I did myself
- Typed and saved every file into the project
- Ran and tested all commands (dotnet build, migrations, dotnet run)
- Debugged and fixed typos/bugs in the generated code
- Tested the full app flow (login, CRUD, logout) manually in the browser
- Made all git commits

## Why this approach
Given limited prior experience with C#/ASP.NET Core and a short deadline, AI assistance
let me focus on understanding the architecture and decisions (auth flow, EF Core,
REST API design) rather than getting stuck on unfamiliar syntax. I can explain the
purpose and behavior of every part of this codebase during the interview.
